# Host Project Rules

Customize this file after installing `codex_with_cc` into a project.

## Defaults
- Do not create or switch branches or worktrees unless the user explicitly asks.
- Do not revert unrelated local changes.
- Keep edits inside the delegated scope unless a small supporting change is necessary.
- Follow the host project's existing style, tooling, and tests.
- If files are changed, run the smallest meaningful verification command for the changed area.
- If the host project has changelog, memory, formatting, lint, build, or test requirements, document them in this file.
